# 图片像素格转换器

一个使用 React + Vite + JavaScript 开发的纯前端图片像素格转换器。用户可以上传本地图片，在浏览器内通过 HTML Canvas 生成像素格效果，并导出 PNG 图片。

> 安全说明：所有图片处理都在浏览器本地完成，不会上传用户图片；项目不包含后端、数据库、登录系统、云存储或支付功能。

## 功能

- 上传一张本地图片并预览原图。
- 在页面中实时查看像素格处理结果。
- 调整像素块大小：4、8、12、16、24、32。
- 选择颜色数量：原图颜色、8 色、4 色、2 色。
- 显示或隐藏网格线。
- 调整网格线颜色和粗细。
- 显示横向格数、纵向格数和总格数。
- 将处理后的图片导出为包含格数标注的 PNG。
- 支持 PWA：构建后可作为静态网页部署，并可添加到手机主屏幕。

## 技术栈

- React
- Vite
- JavaScript
- HTML Canvas
- Web App Manifest + Service Worker

## 本地运行

请先确认本机已安装 Node.js 和 npm。

```bash
npm install
npm run dev
```

启动后，根据终端提示打开本地地址，通常是：

```text
http://localhost:5173/
```

## 手机局域网预览

让手机和电脑连接同一个 Wi-Fi，然后在电脑上运行：

```bash
npm run dev -- --host 0.0.0.0
```

终端会显示局域网地址，例如：

```text
http://192.168.1.23:5173/
```

在手机浏览器中打开这个地址即可预览。手机端页面会自动切换为单列布局，上传区、控制面板、原图预览和像素图预览会依次排列。

> 注意：PWA 安装和 Service Worker 通常需要 HTTPS 或 `localhost`。局域网 HTTP 地址适合测试手机布局和交互；如需完整测试“添加到主屏幕”，建议使用 HTTPS 部署后的地址。

## 构建生产版本

```bash
npm run build
```

构建产物会输出到 `dist/` 目录，可直接作为静态网页部署。

## 预览生产构建

```bash
npm run preview -- --host 0.0.0.0
```

可在电脑或同局域网手机中打开终端提示的地址，检查生产构建效果。

## 部署

本项目不需要服务器端能力，构建后的 `dist/` 目录可以部署到任意静态网站服务，例如：

- GitHub Pages
- Netlify
- Vercel Static Deploy
- Cloudflare Pages
- Nginx/Apache 静态目录

部署步骤通常为：

1. 运行 `npm install` 安装依赖。
2. 运行 `npm run build` 生成 `dist/`。
3. 将 `dist/` 目录作为静态站点根目录发布。
4. 使用 HTTPS 访问部署后的站点，以便 PWA 安装和 Service Worker 正常工作。

## 添加到手机主屏幕

### iPhone / iPad Safari

1. 使用 Safari 打开 HTTPS 部署后的站点。
2. 点击底部分享按钮。
3. 选择“添加到主屏幕”。
4. 确认名称“图片像素格转换器”，点击“添加”。

### Android Chrome

1. 使用 Chrome 打开 HTTPS 部署后的站点。
2. 点击浏览器菜单。
3. 选择“添加到主屏幕”或“安装应用”。
4. 确认安装后，即可像普通 app 一样从主屏幕打开。

## 本地和手机端测试清单

- 上传 JPG、PNG 或 WebP 图片，确认原图预览正常。
- 修改像素块大小，确认像素图和横向/纵向/总格数同步更新。
- 切换颜色数量为“原图颜色 / 8 色 / 4 色 / 2 色”，确认像素图颜色数量变化。
- 开启/关闭网格线，调整网格颜色和粗细。
- 在图片重新渲染时确认导出按钮不可用，渲染完成后再导出。
- 在手机竖屏下确认页面为单列布局，导出按钮清晰可见，预览图不超出屏幕宽度。
- 部署到 HTTPS 静态站点后，确认可以添加到手机主屏幕。
